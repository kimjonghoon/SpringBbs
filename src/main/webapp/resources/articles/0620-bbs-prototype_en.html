Prototype

A prototype is a kind of model that is created at the beginning of development.
The prototype of the Web application is based on the JSP that corresponds to the screen.
Do not implement program logic in prototypes except for those that are needed for testing, or that do not change until the end of development.

I have prepared a Model 1 bulletin board as an example to be implemented in JSPProject.
In Model 1, the client requests the JSP directly.
The following is a list of JSP files required for the example.

The main page and sub page design is the first task of the website prototype.
The main page is the page you see when you visit the website, also called the home page.
The sub page is all pages except the main page.

Main page

I will add the login logic to the prototype.
Therefore, before creating the main page, the following User class is created first.

User.java
package net.java_school.user;

public class User {
    private String email;
    private String passwd;
    private String name;
    private String mobile;
    
    public User() {}

    public User(String email, String passwd) {
        this.email = email;
        this.passwd = passwd;
    }
    
    public User(String email, String passwd, String name, String mobile) {
        this.email = email;
        this.passwd = passwd;
        this.name = name;
        this.mobile = mobile;
    }

    public String getEmail() {
        return email;
    }
    
    public void setEmail(String email) {
        this.email = email;
    }

    public String getPasswd() {
        return passwd;
    }
    
    public void setPasswd(String passwd) {
        this.passwd = passwd;
    }
    
    public String getName() {
        return name;
    }
    
    public void setName(String name) {
        this.name = name;
    }
    
    public String getMobile() {
        return mobile;
    }
    
    public void setMobile(String mobile) {
        this.mobile = mobile;
    }

}

The User class does not have an ID.
I will use email instead of ID.

Login policy

  • When an unauthenticated user visits a page that requires authentication, the user is sent to the login page. At this time, the URL that is intended to be visited first is transmitted to the login page.
  • If the login is successful, the login page should be redirected to the URL that the user want to visit.
  • When the user logs in from the login page, the page is redirected to the first page of the small talk board list.
  • When the user log out, the page is redirected to the homepage.

When creating a common JSP file, the contentType attribute of the page directive must be the same as the containing page.
If it is not the same, it is not converted to a servlet.
Therefore, the contentType of a common JSP file should be: 

<%@ page language="java" contentType="text/html; charset=UTF-8"
    pageEncoding="UTF-8"%>

JSP error handling

Open web.xml.
Add the following before </web-app> in web.xml.

web.xml
<error-page>
    <error-code>403</error-code>
    <location>/error.jsp</location>
</error-page>
<error-page>
    <error-code>404</error-code>
    <location>/error.jsp</location>
</error-page>
<error-page>
    <error-code>500</error-code>
    <location>/error.jsp</location>
</error-page>
<error-page>
    <exception-type>java.lang.Throwable</exception-type>
    <location>/error.jsp</location>
</error-page>

This is a setting that /error.jsp will take care of all major errors.

In the bulletin board program, the pages corresponding to the screen are list.jsp, view.jsp, write.jsp, and modify.jsp.
Of these, except for modify.jsp and all include a loginCheck.jsp file.
modify.jsp requires the logic to check the owner of the post.
EditAccount.jsp, changePasswd.jsp, and bye.jsp among the member programs include the loginCheck.jsp file.
JSP pages that do not show a screen have _proc appended to the filename.

Login Policy

  1. Only authenticated users can use the bulletin board.
  2. If login is successful in login page, the login page be redirected to /bbs/list.jsp?boardCd=smalltalk&curPage=1.
  3. If an unauthenticated user visits a page that requires authentication, it will be redirected to the login page. If the login is successful, the login page will be redirected to the page you originally intended to visit.
  4. When logout is complete, the logout.jsp be redirected to the homepage.
